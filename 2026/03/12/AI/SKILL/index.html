注意事项

  1. 代码生成顺序: 先 make code 生成代码，再开发业务逻辑
  2. 模块通信: 同服通过 agent.Modules 直接调用，跨服通过 HTTP API
  3. 导出方法: 首字母大写，会被 make agent 扫描生成接口
  4. 非导出方法: 首字母小写，仅模块内部使用
  5. 事务处理: 使用 middleware.GoTransaction 包装事务
  6. 异步操作: 使用 middleware.Go 或 middleware.GoWork 执行异步任务
  7. 测试隔离: 使用 mock.User(t) 创建独立测试环境
  8. 命名规范: {Module} 为大驼峰，{module} 为小写下划线
  9. 问题反馈: 遇到不懂的、不熟悉的问题及时反馈给用户，让用户来帮助解答

mock/apis.go 更新说明

创建新模块后，需要在 internal/gameserver/test/mock/apis.go 中添加模块引用，否则测试无法运行。
