2.3 缓存雪崩

问题：大量 key 同时失效，或 Redis 宕机

与击穿的区别：

  • 雪崩：多个 key 同时失效
  • 击穿：单个热点 key 失效

解决方案：

方案 做法
随机 TTL 1800 + rand.Intn(300) 秒
集群高可用 哨兵/集群模式
限流降级 rate.NewLimiter(1000, 100)

第五篇：过期与淘汰

5.1 过期策略

惰性删除：访问时检查过期，删除

定期删除：定时抽查部分 key

Redis 实际：惰性删除 + 定期删除结合

5.2 淘汰策略（8种）

分类 策略 说明
不淘汰 noeviction 内存满返回错误
全部 key allkeys-lru LRU 最近最少使用
allkeys-lfu LFU 最不经常使用
allkeys-random 随机淘汰
带过期时间 volatile-lru 仅过期的 key 中 LRU
volatile-lfu 仅过期的 key 中 LFU
volatile-random 仅过期的 key 中随机
volatile-ttl 淘汰即将过期的

使用建议：

场景 推荐策略
热点数据 + 全量缓存 allkeys-lru
热点数据 + 部分缓存 volatile-lru
不确定访问模式 allkeys-lfu
消息队列 volatile-ttl
